# تشغيل واتساب من جهازك المحلي بدل الـ VPS

هذا المسار يجعل جلسة واتساب والارسال يعملان من جهازك نفسه.

المعنى العملي:
- تمسح QR من جوالك على جهازك المحلي.
- عامل واتساب يعمل على جهازك المحلي.
- الموقع يواصل استخدام نفس طابور Supabase، لكن التنفيذ الفعلي يتم من جهازك.

مهم:
- لا تشغل عامل الـ VPS والعامل المحلي معًا في نفس الوقت.
- الطابور الحالي مشترك، وتشغيل العاملين معًا قد يسبب معالجة متداخلة.
- إذا انتهى اشتراك الـ VPS أو أوقفته، شغل العامل المحلي فقط.

## 1. تجهيز ملف البيئة

انسخ:

```bat
copy .env.local-worker.example .env.local-worker
```

ثم عبئ القيم التالية من نفس إعدادات الموقع:
- `SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

اترك هذه القيم كما هي إذا كنت تريد أن يحل جهازك محل الـ VPS مباشرة:
- `WHATSAPP_WORKER_STATE_SETTING_ID=whatsapp_worker_state`
- `WHATSAPP_WORKER_COMMAND_SETTING_ID=whatsapp_worker_command`

## 2. تثبيت الاعتماديات

إذا لم تكن مثبتة:

```bat
npm install
```

## 3. تشغيل العامل المحلي

شغل الملف:

```bat
run-local-whatsapp-worker.bat
```

أو يدويًا:

```bat
set WORKER_ENV_FILE=.env.local-worker
set WHATSAPP_WORKER_MODE=local
npm run worker:whatsapp
```

## 4. ربط واتساب من جوالك

بعد تشغيل العامل:
- افتح الموقع.
- ادخل نافذة باركود واتساب.
- امسح QR من جوالك.
- بعد نجاح الربط سيظهر أن الواتساب متصل على `الجهاز الحالي`.

## 5. الإرسال

بعد الربط:
- أي إرسال من صفحة الواتساب سيتم عبر جهازك المحلي.
- إذا كان العامل المحلي متوقفًا، ستبقى صفحة الإرسال قادرة فقط على تجهيز روابط واتساب اليدوية.

## 6. إيقاف التشغيل

عند الانتهاء أغلق نافذة العامل المحلي أو أوقف العملية من الطرفية.

## ملاحظة مهمة

إذا كان Chrome أو Edge غير مكتشف تلقائيًا على ويندوز، ضع المسار في:

- `PUPPETEER_EXECUTABLE_PATH`

مثال:

```env
PUPPETEER_EXECUTABLE_PATH=C:\Program Files\Google\Chrome\Application\chrome.exe
```
